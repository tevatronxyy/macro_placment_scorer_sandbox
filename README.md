# Innovus Macro Placement: Immutable Scoring Mechanism

## Overview
The **Immutable Scoring Mechanism** is a core component of the multi-agent macro placement system. It acts as an objective, tamper-proof evaluator that scores the quality of a macro placement layout generated by the AI agents. 

Because foundational AI models can struggle with raw physical design constraints, we guide them using three proxy heuristic metrics. These rules operate on a "penalty" basis—the goal of the iterative layout agent is to minimize these penalties, resulting in a higher overall placement score.

---

## The 3-Metric Scoring Matrix

### 1. Boundary Hugging Penalty
**Objective:** Maximize center-die area for standard cell placement by forcing macros toward the core boundaries.
**Why it matters:** Leaving a contiguous, uniform open space in the center of the die prevents routing congestion and allows for highly optimized standard logic cell placement.

*   **Mechanism:** For every placed macro, the system calculates the absolute distance from its bounding box edges to the four edges of the design core boundary (Top, Bottom, Left, Right).
*   **Calculation:** Only the *minimum* distance (the closest edge) is recorded as a penalty. 
*   **Agent Goal:** Place macros flush against the outer edges (or within the acceptable 1-2um margin) to drive this penalty toward zero.

### 2. Clustering (Padded Union) Penalty
**Objective:** Ensure macros are placed as densely as possible without creating illegal overlaps, reducing overall wirelength.
**Why it matters:** Tightly packed macros shorten dataflow paths and preserve valuable floorplan real estate.

*   **Mechanism:** The system draws a virtual **10um halo (padding)** around every macro. It then performs a Boolean Union (an "AND" process in spatial geometry) to merge all intersecting padded rectangles into a single, complex polygon.
*   **Calculation:** The penalty is the total aggregate area of this merged shape. If macros are placed far apart, their halos do not overlap, resulting in a massive total area. If they are clustered tightly, the halos merge, significantly reducing the total calculated area.
*   **Agent Goal:** Cluster macros close enough that their 10um halos heavily overlap, minimizing the total united area.

### 3. Hierarchy Proximity (HPWL) Penalty
**Objective:** Group macros that belong to the same logical hierarchy (parent module) close together.
**Why it matters:** Macros under the same hierarchy usually share heavy data and control paths. Grouping them physically minimizes routing delays and improves timing performance.

*   **Mechanism:** The system groups all macros based on their parent hierarchy string (e.g., `parent_A/child_B/macro_inst`). 
*   **Calculation:** For each hierarchical group containing more than one macro, the system calculates the **Half-Perimeter Wirelength (HPWL)**. It draws a bounding box that completely encloses all macros in that specific group, and adds the width and height of that box together.
*   **Agent Goal:** Keep macros of the same family directly adjacent to one another to minimize the size of the hierarchical bounding box.

---

## Total Score Calculation

The final placement score is calculated by combining the three penalties using a weighted matrix. 

1.  Each penalty is multiplied by a configurable weight (e.g., $w_1$ for Boundary, $w_2$ for Clustering, $w_3$ for Hierarchy).
2.  The weighted penalties are summed together.
3.  The total penalty is subtracted from a predefined `baseline_worst_case` constant to invert the logic.

**Final Score = `Baseline - ((W1 * Boundary) + (W2 * Clustering) + (W3 * HPWL))`**

The stateless iterative agent will continue adjusting the coordinates and checking this immutable tool until the final score meets the required threshold for a "Good" placement.
